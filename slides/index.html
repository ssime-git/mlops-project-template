Act VI · Pratiques
21 / 27
Principes

Bonnes pratiques

Reproductibilité avant tout
Environnements conteneurisés, dépendances figées, seeds aléatoires fixées.
Versioning systématique
Code, données, modèles et expériences versionnés dès le jour 1.
Tests automatisés
Tests unitaires, d'intégration et de validation des données.
Documentation vivante
README, doc technique, architecture — tenue à jour avec le code.
Sécurité par conception
API sécurisées, secrets management, principe du moindre privilège.
Monitoring dès le début
Ne pas attendre la fin — métriques, alertes, dashboards jour 1.

23 / 27
Production

Mise en production

Déploiement progressif
Canary, blue-green, feature flags — ne jamais déployer en masse.
Rollback immédiat
Chaque déploiement doit pouvoir être annulé en 1 clic.
Observabilité
Logs, métriques, traces — tout doit être mesurable.
